Aplicación Móvil de Seguimiento de Hábitos

Introducción

Este documento detalla la especificación del diseño de una aplicación Android para el seguimiento de hábitos, desarrollada con Kotlin y Jetpack Compose. La aplicación está diseñada para ayudar a los usuarios a gestionar sus hábitos diarios mediante un sistema intuitivo de creación, edición y seguimiento de actividades.

Kotlin Jetpack Compose Firebase Room

Análisis de Requisitos

Requisitos Funcionales

  • Login
    • Registro e inicio de sesión mediante Firebase Authentication
    • Gestión segura de credenciales
  • Gestión de Hábitos
    • CRUD completo de hábitos
    • Programación semanal (L-D)
    • Sistema de recordatorios con AlarmManager
  • Pantalla Home
    • Vista organizada por días
    • Navegación intuitiva a detalles
  • Detalles del Hábito
    • Configuración de nombre
    • Selección de días
    • Configuración de notificaciones

Requisitos No Funcionales

  • Seguridad robusta con Firebase Authentication
  • Alto rendimiento y optimización de recursos
  • UI/UX siguiendo Material Design

Diseño de Datos

La aplicación utiliza una combinación de almacenamiento local y servicios en la nube:

Datos de Aplicación

  • Room Database para hábitos y configuraciones
  • SharedPreferences para preferencias de usuario
  • Integración con AlarmManager para recordatorios

Diseño de la Interfaz de Usuario

La interfaz se construye completamente con Jetpack Compose, implementando:

  • Pantalla de Login
  • Home con lista de hábitos
  • Editor de hábitos
  • Sistema de notificaciones

Arquitectura del Sistema

La aplicación implementa MVVM y Clean Architecture:

  • Model: Room + Retrofit
  • View: Jetpack Compose
  • ViewModel: Android Architecture Components

Componentes Principales

  • Frontend en Kotlin con Jetpack Compose
  • Backend con Firebase Authentication
  • Inyección de dependencias con Dagger Hilt
